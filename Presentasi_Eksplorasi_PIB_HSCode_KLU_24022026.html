07 — Instruksi Kerja
Panduan untuk Business Analyst
1

Pemetaan HS Code ke Klaster

Identifikasi seluruh POS_TARIF_HS beserta KD_KELOMPOK dan NM_DETIL yang relevan. Filter hanya tahun 2022–2024. Perhatikan kode "Lain-lain" (62,2% data) dan baris NaN (18.947 baris tanpa kelompok).

2

Analisis Sinkronisasi HS × KLU

Kewajaran: Apakah KLU WP wajar untuk barang yang diimpor? Dispersi: Berapa banyak KLU unik per HS Code? (>100 = perhatian khusus). Dominan vs Outlier: KLU mayoritas vs KLU anomali — prioritaskan outlier bernilai tinggi.

3

Profiling per HS Code

Analisis: konsentrasi importir (jml_npwp sedikit tapi nilai tinggi), rasio PPh/PPN anomali, tren 3 tahun (naik/turun signifikan), HS "Lainnya" yang mendominasi nilai, dan KLU tidak wajar bernilai besar.

4

Cross-check Antar Klaster

Apabila ditemukan HS Code yang seharusnya masuk klaster lain, koordinasikan dengan tim klaster terkait untuk menghindari duplikasi analisis.

09 — Output yang Diharapkan
Deliverables per Klaster
Setiap tim klaster menghasilkan 5 output berikut sebagai prasyarat sebelum tahap pemodelan risiko.
1
Daftar HS Code Final
Kode HS yang masuk scope klaster, beserta justifikasi — termasuk keputusan untuk kode "Lainnya" dan baris NaN.
2
Profil Ringkasan per HS Code
Tren 3 tahun (2022→2024), konsentrasi importir, nilai pajak, dan dispersi KLU per kode HS.
3
Matriks Sinkronisasi HS × KLU
Daftar kombinasi wajar dan kombinasi anomali untuk setiap HS Code prioritas dalam klaster.
4
Rekomendasi HS Code Prioritas
Untuk pendalaman risk event: Mispricing, Misdeclaration, API-P Abuse, Artificial Loss, dll.
5
Catatan Kebutuhan Data Tambahan
Data yang perlu disiapkan Data Engineer untuk analisis lanjutan — misalnya data per-NPWP, per-transaksi, per-negara asal dari 70 juta baris raw, atau data SPT dan e-Faktur untuk fase berikutnya.
